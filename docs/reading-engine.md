# 鑑定エンジン設計

## 概要

ホシミ占星館の鑑定は、**クライアント側の計算エンジン**と**サーバー側の鑑定文生成**の2層で構成されています。

命式の算出・タロットのカード選択・暦注計算はすべてクライアント側で行い、サーバーはその結果を元に鑑定文を生成します。

## 算命学エンジン

### 計算フロー

```
生年月日入力
  │
  ├── 1. 万年暦テーブルから該当年月日を検索
  │
  ├── 2. 年柱算出（年干 + 年支）
  ├── 3. 月柱算出（月干 + 月支）※節入り日を考慮
  ├── 4. 日柱算出（日干 + 日支）
  │
  ├── 5. 通変星導出（日干と各干の関係から算出）
  ├── 6. 十二運導出（日干と各支の関係から算出）
  │
  ├── 7. 五行バランス計算
  │     ├── 天干の五行分類（木火土金水）
  │     ├── 地支の蔵干から五行を抽出
  │     └── 各五行のスコアを集計
  │
  └── 8. 命式表の構築
        ├── 年柱: 天干 + 地支 + 通変星 + 十二運
        ├── 月柱: 天干 + 地支 + 通変星 + 十二運
        └── 日柱: 天干 + 地支 + 通変星 + 十二運
```

### クライアント側で行う理由

- ネットワーク不要で即座に結果表示（UX向上）
- サーバー負荷を最小化
- 計算ロジック自体は決定論的（同じ入力 → 同じ出力）

## タロットエンジン

### スプレッド対応

| スプレッド | カード枚数 | 用途 |
|-----------|-----------|------|
| **ワンオラクル** | 1枚 | 今日のメッセージ、シンプルな問い |
| **スリーカード** | 3枚 | 過去・現在・未来 |
| **ケルト十字** | 10枚 | 深い分析（状況・障害・過去・未来・意識・深層心理等） |

### カード選択の流れ

```
テーマ入力
  │
  ▼
シャッフルアニメーション
  │ ユーザーがタップで停止
  ▼
カットアニメーション
  │ ユーザーがタップで停止
  ▼
カード配置（伏せ状態）
  │
  ▼
1枚ずつタップでめくり
  │ めくりアニメーション
  │ 正位置 / 逆位置をランダムで決定
  ▼
全カードめくり完了 → 鑑定文生成リクエスト
```

## 運勢カレンダーエンジン

日ごとの暦注（吉日・凶日・六曜など）を計算:

| 項目 | 計算方法 |
|------|----------|
| **日の干支** | 基準日からの日数差で60干支を巡回 |
| **六曜** | (旧暦月 + 旧暦日) % 6 で算出 |
| **天赦日** | 季節の天干地支の組み合わせで判定 |
| **一粒万倍日** | 月支と日支の特定の組み合わせで判定 |
| **大安・仏滅等** | 六曜から判定 |

## 鑑定文生成（サーバー側）

### 2段階の鑑定

| 鑑定者 | モデル | トリガー | コスト |
|--------|--------|----------|--------|
| **ウララ（見習い）** | Gemini Flash Lite | 広告視聴 | 無料（広告収益） |
| **ホシミ先生（師匠）** | Gemini Pro | クレジット消費 | 2クレジット/回 |

### キャッシュ戦略（算命学のみ）

同じ生年月日の命式に対する鑑定文は、月単位でキャッシュ:

```
鑑定リクエスト
  │
  ├── cache_key = SHA256(プロンプト内容)
  │
  ├── キャッシュ検索
  │     ├── ヒット + 同月 → キャッシュから返却（クレジット消費なし）
  │     └── ミス or 別月 → Gemini API呼び出し → キャッシュ保存
  │
  ▼
レスポンス
```

月が変わるとキャッシュが無効になる設計。運勢は月ごとに変化するという占い的な観点と、鮮度を保つ技術的な観点の両方を満たしています。

### エラー時のクレジット返還

```
Gemini API呼び出し失敗
  └── クレジットを自動返還

出力フィルタリングでブロック
  └── クレジットを自動返還
```

ユーザーに非がないエラーの場合、消費したクレジットを自動返還します。
