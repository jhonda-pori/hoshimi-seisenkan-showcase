## 認証設計

Firebase Authentication は使用せず、独自のトークンベース認証を採用:

```
ユーザー登録
  │
  ├── device_id + username を送信
  │
  ▼
サーバーがUUIDでuser_id生成
  │
  ├── auth_token（64文字hex）を発行
  ├── 有効期限: 90日
  │
  ▼
以降のリクエストは X-Auth-Token ヘッダーで認証
```

### 認証の多重チェック

1. **X-Auth-Token** — トークンの有効性と有効期限を検証
2. **device_id照合** — リクエストボディのdevice_idとDB上のユーザーが一致するか検証
3. **リソース所有権** — 鑑定結果へのアクセス時、結果のuser_idとリクエスト元ユーザーが一致するか検証

## データモデル

### サーバー側 (SQLite)

| テーブル | 用途 |
|----------|------|
| **users** | ユーザー情報（username, device_id, birth_date, ad_free） |
| **credits** | クレジット残高 |
| **credit_history** | クレジット増減の履歴 |
| **fortune_results** | 算命学鑑定結果（命式データ + 鑑定文） |
| **tarot_results** | タロット結果（テーマ, スプレッド, カード + 鑑定文） |
| **reading_cache** | 鑑定文キャッシュ（算命学、月単位で有効） |
| **chat_messages** | 追加質問チャットの会話履歴 |
| **purchases** | 購入済みトランザクション（冪等性保証） |
| **user_auth_tokens** | 認証トークン |

### クライアント側 (sqflite)

ローカルDBに鑑定結果・履歴をキャッシュし、オフラインでも過去の結果を閲覧可能。

## 技術選定の理由

| 技術 | 選定理由 |
|------|----------|
| **Flutter** | Android/iOS両対応。リッチなアニメーション（タロットシャッフル等）の実装に適する |
| **Riverpod** | 型安全な状態管理。Providerの依存関係を宣言的に定義可能 |
| **Python / Flask** | 軽量で十分な機能。算命学の暦データ処理に適する |
| **SQLite (WAL)** | 個人サーバー運用に最適。WALモードで読み書きの並行性を確保 |
| **Gemini API (Vertex AI)** | 日本語の自然な文章生成。Vertex AI経由でサービスアカウント認証 |
| **Cloudflare Tunnel** | オリジンサーバーのIP秘匿。無料プランで十分な保護 |
| **sqflite** | Flutter標準のSQLiteラッパー。オフライン対応に必須 |
