## 認証設計

Firebase Authentication は使用せず、独自のトークンベース認証を採用:

```
ユーザー登録
  │
  ├── デバイス識別子 + ユーザー名を送信
  │
  ▼
サーバーが認証トークンを発行（有効期限付き）
  │
  ▼
以降のリクエストはヘッダーにトークンを付与して認証
```

### 認証の多重チェック

1. **トークン検証** — トークンの有効性と有効期限を検証
2. **デバイス照合** — リクエスト元のデバイスとDB上のユーザーが一致するか検証
3. **リソース所有権** — 鑑定結果へのアクセス時、リクエスト元ユーザーがデータの所有者か検証

## データモデル

### サーバー側 (SQLite)

主要なデータ領域:

- **ユーザー管理** — アカウント情報・認証トークン
- **クレジット** — 残高管理・増減履歴
- **鑑定結果** — 算命学・タロットの鑑定データと鑑定文
- **鑑定文キャッシュ** — 同一条件の鑑定文を月単位でキャッシュ
- **チャット履歴** — 追加質問の会話ログ
- **購入記録** — 課金トランザクションの冪等性保証

### クライアント側 (sqflite)

ローカルDBに鑑定結果・履歴をキャッシュし、オフラインでも過去の結果を閲覧可能。

## 技術選定の理由

| 技術 | 選定理由 |
|------|----------|
| **Flutter** | Android/iOS両対応。リッチなアニメーション（タロットシャッフル等）の実装に適する |
| **Riverpod** | 型安全な状態管理。Providerの依存関係を宣言的に定義可能 |
| **Python / Flask** | 軽量で十分な機能。算命学の暦データ処理に適する |
| **SQLite (WAL)** | 個人サーバー運用に最適。WALモードで読み書きの並行性を確保 |
| **Gemini API (Vertex AI)** | 日本語の自然な文章生成。Vertex AI経由でサービスアカウント認証 |
| **Cloudflare Tunnel** | オリジンサーバーのIP秘匿。無料プランで十分な保護 |
| **sqflite** | Flutter標準のSQLiteラッパー。オフライン対応に必須 |
